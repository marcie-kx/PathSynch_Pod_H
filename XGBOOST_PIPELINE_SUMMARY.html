1) Pipeline in 4 steps

  1. Load raw data and clean date fields.
  2. Aggregate daily targets and ad metrics.
  3. Create lag/rolling/calendar feature table.
  4. Run recursive rolling backtest, then 30-day forecast.

2) Why this is trustworthy

  • Evaluation is recursive (multi-step), matching real forecast usage.
  • Lag features use raw history to avoid train-vs-forecast scale mismatch.
  • Same-day ad leakage was removed (previous-day and lagged ad features only).
  • Forecast interval is an empirical forecast interval built from the out-of-fold error quantile from backtest.

4) Main feature groups used

  • Calendar: day/week/month/quarter, weekend, month-end, holiday flags.
  • Autoregressive lags: 1, 2, 3, 7, 14, 21, 28, 35, 56.
  • Rolling stats: mean/std/min/max over 3/7/14/28-day windows.
  • Ad lags: spend, CTR, CPC on previous day and 7/14-day lags.
  • Interaction features: lag differences and short-vs-long rolling differences.

5) Delivery files

  • Code: utils.py, build_features.py, backtest.py, forecast_30d.py
  • Output: metrics.json and forecast_30d.csv
